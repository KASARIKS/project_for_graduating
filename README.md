# Файлы для итогового задания

Описание проекта: Проект был создан для прохождения финального задания. Проект является туду-листом. Можно выполнять добавление, удаление, изменение задач, а также установку повторения на определённые интервалы.

Список выполненных задач со звездочкой(Выполнены все задания, но на всякий случай их список снизу):
- Шаг 1. Прослушивание переменной окружения TODO-PORT, и запуск порта на порте, указанном в этой переменной.
- Шаг 2. Прослушивание переменной окружения TODO-DBFILE, и создания файла базы данных в месте, указанном в этой переменной(путь к файлу).
- Шаг 3. Опция w и m.
- Шаг 5. Опциональный поиск(по дате, слову и просто).
- Шаг 8. Аутентификация, dockerfile.

Инструкция по запуску кода локально(Мог указать неправильно, напишите если не так):
- Запустить программу можно собрав exe через go build.
- Программа запускается по адресу localhost:8080(порт 8080, может отличаться, если указать значение в переменной окружения TODO-PORT).
- Если было указано значение в переменно TODO-PASSWORD, то нужно будет авторизоваться с этим паролем. Доступ открыт в течение 15 минут.

Инструкция по запуску тестов:
Файл settings.go должен выглядеть так:
package tests

var Port = 7540
var DBFile = "../scheduler.db"
var FullNextDate = true
var Search = true
var Token = `123`
Чтобы запустить тесты, нужно ввести команду go test ./tests(при этом нужно находиться внутри каталога проекта).

Инструкция по сборке и запуску проекта через докер:
Проект запускается с помощью команды docker run -d -p 7540:7540 graduate_project:v1.
(в команде докера значения 7540:7540, возможно понадобится заменить на 8080:8080 или на другие, зависит от значения TODO-PORT)

В директории `tests` находятся тесты для проверки API, которое должно быть реализовано в веб-сервере.

Директория `web` содержит файлы фронтенда.
